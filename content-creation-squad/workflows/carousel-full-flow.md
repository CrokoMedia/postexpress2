# Workflow: Criação Completa de Carrossel

**Duração estimada**: 10-15 minutos
**Frameworks aplicados**: Copy Científico (principal), Posicionamento, Value Stack, Adaptação Cultural, Atenção
**Output**: Carrossel completo pronto para produção visual

---

## ⚠️ RESTRIÇÃO CRÍTICA PARA TODO O WORKFLOW

**Durante TODAS as fases, o sistema deve:**
- ❌ NUNCA inventar números, métricas, cases ou resultados fictícios
- ❌ NUNCA criar "provas sociais" ou testemunhos sem dados reais do usuário
- ✅ Usar APENAS dados que o usuário fornecer explicitamente
- ✅ Marcar claramente exemplos como [TEMPLATE] ou [EXEMPLO DIDÁTICO]
- ✅ Deixar placeholders [INSERIR DADO REAL] quando faltar informação

**Se o sistema precisar de um número/case que não foi fornecido:**
→ Deve PARAR e pedir explicitamente ao usuário: "Preciso que você forneça [X dado real]"
→ NÃO deve continuar inventando dados

---

## 🎯 Fase 1: BRIEFING (2 min)

### Coleta de Informações

**Perguntas obrigatórias**:
1. Qual o **OBJETIVO** principal?
   - [ ] Educar/ensinar
   - [ ] Vender produto/serviço
   - [ ] Construir autoridade
   - [ ] Gerar viralização

2. Qual a **PLATAFORMA**?
   - [ ] Instagram
   - [ ] LinkedIn
   - [ ] Ambas

3. Quem é a **AUDIÊNCIA**?
   > Exemplo: "Empreendedores digitais iniciantes, 25-40 anos, buscando primeira venda online"

4. Qual a **MENSAGEM PRINCIPAL** em 1-2 frases?
   > Exemplo: "Ensinar 5 erros de copy que matam conversão e como corrigir cada um"

5. Qual o **CTA** desejado?
   - [ ] Salvar post
   - [ ] Comentar
   - [ ] Compartilhar
   - [ ] Acessar link
   - [ ] Enviar DM
   - [ ] Outro: ______

**Informações opcionais**:
- Tom preferido: [educacional/provocativo/inspirador/outro]
- Número de slides: [6-10, padrão: 8]
- Palavras-chave importantes: _______
- Evitar: _______

---

## 🎨 Fase 2: ESTRATÉGIA (3 min)

### Mente Ativa: Seth Godin

**Objetivo**: Definir ângulo estratégico e estrutura

### Prompt para Seth:
```
Baseado neste briefing:
- Objetivo: [objetivo do usuário]
- Audiência: [audiência do usuário]
- Mensagem: [mensagem do usuário]
- Plataforma: [plataforma]

Defina:

1. **Ângulo estratégico**
   Qual perspectiva única tomar neste tema?
   O que fará este carrossel ser "remarkable" (digno de ser comentado)?

2. **Tom de voz**
   Qual tom será mais efetivo?
   - Educacional/professoral
   - Conversacional/amigável
   - Provocativo/polêmico
   - Inspiracional/motivacional
   - Outro: ______

3. **Estrutura ideal**
   Quantos slides?
   Qual progressão narrativa?
   Qual fórmula usar? (problema-solução / lista / framework / story / vendas / controvérsia)

4. **Hook strategy**
   Qual tipo de hook usar no slide 1?
   - Pergunta provocativa
   - Número específico
   - Opinião polêmica
   - Promessa grande
   - Revelação de segredo

5. **Viralidade**
   Que elemento incluir para aumentar saves/shares?
```

### Output Esperado:
- Ângulo escolhido
- Tom definido
- Estrutura (fórmula + número de slides)
- Tipo de hook
- Elemento viral

---

## ✍️ Fase 3: COPYWRITING (5 min)

### Mente Ativa: Eugene Schwartz

**Objetivo**: Escrever copy completo de cada slide

### Primeiro: Determinar Awareness Stage

```
Qual awareness stage da audiência?
- Unaware (não sabe do problema)
- Problem Aware (sabe do problema, não da solução)
- Solution Aware (conhece soluções, não a sua)
- Product Aware (conhece seu produto)
- Most Aware (pronto para comprar)
```

### Prompt para Eugene:

```
Usando a estratégia definida:
[copiar output do Seth]

E considerando awareness stage: [ESTÁGIO]

Escreva:

1. **5 OPÇÕES DE HOOK (Slide 1)**
   Baseado em awareness stage e tipo definido
   Cada hook deve:
   - Parar o scroll
   - Ser específico (números, não genérico)
   - Criar gap de curiosidade
   - Prometer benefício claro

2. **COPY COMPLETO (Slides 2-9)**
   Para cada slide:
   - Título principal (grande, bold)
   - Subtexto (1-2 linhas, se necessário)
   - Indicação de elemento visual

3. **SLIDE 10 (CLOSER)**
   - Reforço da mensagem
   - Pergunta ou pedido de interação
   - Branding sutil

4. **CAPTION**
   - Hook expandido (primeira frase)
   - Desenvolvimento (2-3 parágrafos)
   - CTA final

Princípios a seguir:
- Especificidade > generalização
- Benefício claro em cada slide
- Progressão emocional → racional
- Uma ideia por slide
- Máximo 2-3 linhas por slide
```

### Output Esperado:
- 5 opções de hook testadas
- Copy completo slides 1-10
- Caption

---

## 💰 Fase 4: OTIMIZAÇÃO DE VENDAS (3 min)

### Mente Ativa: Alex Hormozi

**Condicional**: Apenas se objetivo = Vender

### Prompt para Alex:

```
Revise este carrossel de vendas:
[copiar copy do Eugene]

Otimize usando framework $100M Offers:

1. **Value Stack**
   O que está incluído na oferta?
   Como apresentar valor total vs preço?
   Listar componentes com valores individuais

2. **Scarcity & Urgency**
   Por que agir AGORA?
   - Escassez real (vagas limitadas?)
   - Urgência real (deadline?)
   - Consequência de esperar

3. **Guarantee (Reversal de Risco)**
   Como reduzir risco percebido?
   - Garantia de devolução?
   - Garantia de resultado?
   - Prova social forte?

4. **Price Anchoring**
   Como apresentar o preço?
   - Comparação (valor total vs preço)
   - Fracionamento (12x de R$ X)
   - Custo de NÃO ter (oportunidade perdida)

5. **Bonus Stack**
   Que bônus adicionar para desequilibrar a balança?
   Bônus devem:
   - Resolver objeções
   - Aumentar valor percebido
   - Ter valor real (não encher linguiça)

Reescreva slides 5-7 (solução/oferta/CTA) otimizados.
```

### Output Esperado:
- Value stack detalhado
- Urgência/escassez definida
- Garantia formulada
- Apresentação de preço otimizada
- Bonus stack (se aplicável)

---

## 🇧🇷 Fase 5: LOCALIZAÇÃO BR (2 min)

### Mente Ativa: Thiago Finch

**Condicional**: Se audiência for brasileira

### Prompt para Thiago:

```
Adapte este copy para audiência brasileira:
[copiar copy atual]

Ajustes necessários:

1. **Linguagem**
   - Trocar termos em inglês por português quando melhor
   - Usar linguagem coloquial BR apropriada
   - Ajustar formalidade ao contexto

2. **Referências Culturais**
   - Adaptar exemplos para realidade brasileira
   - Usar referências que ressoam no BR
   - Ajustar valores/métricas para R$

3. **Gatilhos Mentais BR**
   - Quais gatilhos funcionam melhor aqui?
   - Urgência/escassez: como apresentar?
   - Prova social: formato preferido no BR?

4. **Tom e Contexto**
   - Ajustar para mercado brasileiro
   - Considerar diferenças culturais
   - Manter autenticidade

Reescreva copy localizado.
```

### Output Esperado:
- Copy adaptado para BR
- Exemplos/referências locais

---

## 🎨 Fase 6: PLANEJAMENTO VISUAL (2 min)

### Objetivo: Especificar layout de cada slide

Para cada slide, definir:

```markdown
### SLIDE [N]

**Copy**:
[Texto do slide]

**Hierarquia Visual**:
- Título: [Tamanho/peso]
- Subtexto: [Tamanho/peso]

**Elementos Visuais**:
- Fundo: [Cor sólida / gradiente / imagem]
- Ícones: [Quais/onde]
- Formas: [Se houver]
- Imagem: [Se houver]

**Paleta**:
- Cor primária: [#HEX]
- Cor secundária: [#HEX]
- Texto: [#HEX]

**Anotações**:
[Qualquer nota especial]
```

### Especificações Técnicas

**Instagram**:
- Formato: 1080x1080px
- Fonte mínima: 40pt
- Máximo 2-3 linhas

**LinkedIn**:
- Formato: 1080x1080px
- Fonte mínima: 36pt
- Tom mais profissional

---

## ✅ Fase 7: REVIEW FINAL (2 min)

### Checklist de Qualidade

**Conteúdo**:
- [ ] Hook passa no "scroll stop test"?
- [ ] Cada slide = uma ideia clara?
- [ ] Progressão lógica entre slides?
- [ ] CTA específico e acionável?
- [ ] Zero typos/erros de português?
- [ ] Números e fatos verificados?

**Estrutura**:
- [ ] Slide 1 irresistível?
- [ ] Slides 2-7/8 entregam promessa do hook?
- [ ] Slide 9 tem CTA claro?
- [ ] Slide 10 incentiva interação?
- [ ] Caption complementa (não duplica)?

**Visual (planejado)**:
- [ ] Texto legível em mobile?
- [ ] Máximo 2-3 linhas por slide?
- [ ] Cores com contraste adequado?
- [ ] Branding consistente?
- [ ] Hierarquia visual clara?

**Otimização**:
- [ ] Alt text planejado?
- [ ] Horário de postagem definido?
- [ ] Métricas de acompanhamento definidas?

## 🔄 Workflows Alternativos

### Quick Carousel (5 min)
Pula fases 4 e 5, usa template pronto.

### Batch Production (30 min)
Cria 4-5 carrosséis de uma vez seguindo temas relacionados.

### Optimize Existing (3 min)
Apenas fase 3 (Eugene) + 7 (review) para otimizar carrossel existente.
