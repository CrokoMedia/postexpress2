1. VISÃO GERAL
================================================================================

1.1 PROBLEMA
------------
Infoprodutores, experts e empresas enfrentam desafios críticos na criação de conteúdo:

• Perdem tempo excessivo pensando e criando conteúdo
• Não conseguem manter consistência de postagens
• Falta escala na produção de conteúdo
• Dificuldade em construir autoridade de forma sistemática
• Engajamento baixo e poucos seguidores
• Conteúdo não converte em vendas


1.2 SOLUÇÃO
-----------
Croko Labs é um sistema interno da Pazos Media que automatiza 80% do processo
de criação de conteúdo, utilizando IA com metodologias comprovadas de marketing
para gerar carrosséis de alta conversão em escala.


1.3 PROPOSTA DE VALOR
---------------------
• Escala: Atender mais clientes sem aumentar equipe
• Qualidade Consistente: IA com metodologia das "mentes" garante padrão
• Margem Maior: Custo por cliente reduzido, lucro aumentado
• Velocidade: Entrega mais rápida = cliente mais satisfeito
• Economia: 70-80% de redução em custos de renderização (vs fluxo tradicional)


1.4 PÚBLICO-ALVO
----------------
• Infoprodutores
• Experts e coaches
• Empresas que precisam de presença digital


1.5 MODELO DE NEGÓCIO
---------------------
Done-for-you: Ferramenta interna da Pazos Media. A equipe opera o sistema e
entrega os conteúdos prontos. O cliente visualiza, edita e aprova conteúdos
através de um portal dedicado com editor visual interativo.

3.2 POR QUE APIFY?
------------------
O scraping externo via Apify foi escolhido para proteger as contas dos clientes.
A integração direta com Instagram pode causar queda de engajamento e riscos de
bloqueio da conta.


3.3 CUSTOS ESTIMADOS (Scraping)
-------------------------------
• Instagram: ~$0.50-1.00 por perfil (50-100 posts)
• TikTok: ~$0.30-0.60 por perfil
• YouTube: ~$0.40-0.80 por canal
• Transcrição (Whisper): $0.006/minuto (~$0.60 por vídeo de 10min)


3.4 CAMPO CRÍTICO: profilePicUrlHD
----------------------------------
O scraper Apify retorna dois campos de foto de perfil:

• profilePicUrl: 150x150px (baixa resolução) - NÃO USAR
• profilePicUrlHD: 320x320px (alta definição) - USAR ESTE ✅

PROCESSAMENTO DA FOTO:
1. Apify scraper extrai profilePicUrlHD do Instagram
2. Backend faz upload imediato no Cloudinary (evita CORS + garante persistência)
3. Cloudinary otimiza: resize 200x200px, crop fill, quality auto
4. URL do Cloudinary é salva em clientes.foto_url
5. URL original do Instagram salva em clientes.foto_url_instagram (backup)

MOTIVOS DO UPLOAD NO CLOUDINARY:
✅ URLs do Instagram têm CORS restritivo
✅ URLs do Instagram podem expirar
✅ Cloudinary oferece CDN global (performance)
✅ Permite transformações (resize, crop, otimização)
✅ Cache automático

EXEMPLO DE PROCESSAMENTO:
```javascript
// Input do Apify
{
  "profilePicUrlHD": "https://instagram.com/.../320x320.jpg"
}

// Upload no Cloudinary
const uploaded = await cloudinary.uploader.upload(
  profilePicUrlHD,
  {
    folder: 'post-express/profile-pics',
    public_id: `cliente_${username}`,
    transformation: [
      { width: 200, height: 200, crop: 'fill' },
      { quality: 'auto:best' }
    ]
  }
);

// Salvar no Supabase
{
  "foto_url": "https://res.cloudinary.com/.../cliente_usuario.png",
  "foto_url_instagram": "https://instagram.com/.../320x320.jpg"
}
```

7.5 PROCESSO DE RENDERIZAÇÃO
----------------------------
1. Cliente clica "Aprovar para Renderização" no Editor
2. Backend recebe JSON final do carrossel (todos os slides)
3. Para cada slide:
   a) CloudinaryExporter converte JSON → URL de transformação
   b) Cloudinary API é chamada para renderizar PNG
   c) Webhook retorna quando imagem está pronta
   d) URL da imagem é salva no Supabase
4. Status do carrossel muda para "Renderizado"
5. Cliente é notificado (email + notificação in-app)


7.6 BATCH PROCESSING
--------------------
Para otimizar, renderizamos em batches de 5 slides por vez:
• Reduz tempo total (paralelização)
• Evita rate limiting da Cloudinary API
• Progress bar para o cliente (20%, 40%, 60%...)

• API service_role_key bypassa RLS (backend)

9.4 FLUXO DE APROVAÇÃO (NOVO)
-----------------------------

1. NOTIFICAÇÃO
   └─ Cliente recebe email: "Novo carrossel disponível"

2. ACESSO AO PORTAL
   └─ Cliente faz login → Vê lista de carrosséis
   └─ Status: 🟡 Rascunho

3. PREVIEW INICIAL
   └─ Cliente clica "Visualizar"
   └─ Vê preview do JSON no template
   └─ Decide: "Editar" ou "Aprovar direto" ou "Solicitar Ajustes IA"

4A. CAMINHO: EDITAR (mais comum)
   └─ Cliente clica "Editar no Canvas"
   └─ Editor Visual abre (full-screen)
   └─ Status muda para: ✏️ Em Edição
   └─ Cliente edita 10-30min
   └─ Auto-save contínuo (a cada 2s)
   └─ Cliente clica "Aprovar para Renderização"
   └─ Status muda para: ⏳ Aguardando Render

4B. CAMINHO: APROVAR DIRETO (raro)
   └─ Cliente clica "Aprovar sem Editar"
   └─ Status muda para: ⏳ Aguardando Render

4C. CAMINHO: AJUSTES IA (ocasional)
   └─ Cliente clica "Solicitar Ajustes"
   └─ Preenche formulário: "O que mudar?"
   └─ Squad Criação re-gera copy
   └─ Volta ao passo 3

5. RENDERIZAÇÃO (Automática)
   └─ Backend pega JSON final
   └─ Status muda para: 🎨 Renderizando (0%)
   └─ Cloudinary processa slides
   └─ Progress atualiza: 20%, 40%, 60%, 80%, 100%
   └─ Status muda para: 🟢 Renderizado

6. DOWNLOAD E PUBLICAÇÃO
   └─ Cliente vê imagens finais
   └─ Faz download (ZIP com 10 PNGs)
   └─ Publica no Instagram/LinkedIn
   └─ Marca como "Publicado" no portal
